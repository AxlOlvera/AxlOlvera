## Filosofía de desarrollo

Tres principios que guían cada decisión técnica:

**Legibilidad sobre brevedad.**  
El código que no necesita explicación es el código bien escrito.

**Contratos claros entre capas.**  
Cada módulo conoce su responsabilidad y sus límites. La complejidad se contiene, no se distribuye.

**Documentación como parte del entregable.**  
Un sistema sin documentación es un sistema que solo su autor puede mantener.
